Users Collection

Single collection with role-based inheritance. ShelterSeeker, Provider, Verificator, Admin share the same collection distinguished by role field.

Listings Collection

Created by verified Providers. Contains embedded review subdocs and manual_blocked_dates. Photos stored as base64 or URLs.

Bookings Collection

Links Seeker to Listing. Status flow: PENDING → ACCEPTED/REJECTED/CANCELLED. Accepted bookings auto-block calendar dates.

Reviews Collection

Separate collection for booking reviews (1:1 with Booking). Seeker can create/update review for accepted bookings only.

Notifications Collection

Push notifications for booking events, verification status changes, and general messages. Linked to any user by user_id.
